# Paper Analysis: CrossSplit: Mitigating Label Noise Memorization through Data Splitting

**Source:** https://arxiv.org/pdf/2212.01674  
**Date analyzed:** 2026-04-11

---

## 1. Summary

**Title:** CrossSplit: Mitigating Label Noise Memorization through Data Splitting  
**Authors:** Jihye Kim, Aristide Baratin, Yan Zhang, Simon Lacoste-Julien  
**Abstract:** Proposes CrossSplit, a dual-network training procedure that splits the labeled dataset into two disjoint halves D1 and D2. Each network is trained on its half using soft labels from the peer network (cross-split label correction) plus the other half as unlabeled data for semi-supervised learning. Since neither network sees the noisy labels its peer is correcting, memorization of wrong labels is prevented. Outperforms UNICON on CIFAR-10/100 (most noise levels), extreme noise on CIFAR-100 (92%, 95%), and mini-WebVision (+0.88%).

**Key contributions:**
- Data splitting as anti-memorization mechanism: peer network cannot have memorized the label it is correcting, unlike all prior label correction methods
- Cross-split label correction: si = βi·ŷ_peer,i + (1-βi)·yi where βi = γ(JSD_norm(ŷ_peer,i, yi) - 0.5) + 0.5; class-balancing via per-class JSD normalization (adapted from UNICON)
- Cross-split SSL: N1 trained on D1 with D2 as unlabeled; N2 on D2 with D1 as unlabeled; uses FixMatch + MixUp + contrastive loss (same as UNICON)
- No sample selection: bypasses the clean/noisy split decision that all DivideMix/UNICON-style methods make; soft correction throughout

**Methodology:**  
PreAct ResNet18 for CIFAR and Tiny-ImageNet; ResNet18 for mini-WebVision. Cosine annealing LR for CIFAR, multi-step for WebVision. Warmup: 10 epochs CIFAR-10, 30 epochs CIFAR-100, 1 epoch WebVision.

**Main results:**  
CIFAR-10 Sym-50%: CrossSplit 96.3% vs UNICON 95.6%; Sym-90%: 91.3% vs 90.8%; Asym-40%: 96.0% vs 94.1%. CIFAR-100 Sym-20%: 79.9% vs 78.9%; Sym-90%: 52.4% vs 44.8%; Asym-40%: 76.8% vs 74.8%. CIFAR-100 Sym-92%: 46.25% vs 32.08%. CIFAR-100 Sym-95%: 29.97% vs 18.14%. mini-WebVision top-1: 78.48% vs UNICON 77.60%. CIFAR-10 Sym-95%: CrossSplit 62.73% vs UNICON 80.82% (failure case).

## 2. Experiment Critique

**Design:**  
Comprehensive evaluation across 4 datasets, symmetric+asymmetric+extreme noise levels, with memorization analysis (training accuracy on clean/noisy samples separately) validating the core mechanism. Ablation study removes individual components (data splitting, class-balancing, label correction) to quantify contributions. T-SNE visualizations show better class separation.

**Statistical validity:**  
3 runs per experiment, mean ± std reported. Last-10-epoch averaging for stable estimates.

**Online experiments (if any):**  
None.

**Reproducibility:**  
Code available on GitHub (SAITPublic/CrossSplit). Training hyperparameters fully documented in Table 9.

**Overall:**  
Strong contribution. The anti-memorization mechanism is conceptually clean and empirically validated. Key limitations: CIFAR-10 extreme noise (>92%) failure is significant; Tiny-ImageNet Sym-50% slight degradation vs UNICON; CIFAR-100 Sym-50% underperforms UNICON (75.7% vs 77.6%). The label correction ablation reveals a 20.63% drop without it on CIFAR-100 Sym-90%, confirming it is the core component.

## 3. Industry Contribution

**Deployability:**  
Moderate. Requires training two networks on dataset halves (same total compute as DivideMix/UNICON but no GMM fitting). The FixMatch + MixUp + contrastive loss pipeline has the same complexity as UNICON. Data splitting is a simple preprocessing step.

**Problems solved:**  
In proxy-label learning: the peer network correction mechanism is directly applicable when proxy labels are generated by a model that could memorize them. Cross-split label correction prevents the label generator from reinforcing its own errors during training.

**Engineering cost:**  
Moderate. Dual-network training with FixMatch, MixUp, and contrastive loss. Same complexity as UNICON. Additional class-balancing JSD normalization is straightforward.

---

## 4. Novelty vs. Prior Work

**Paper's claimed novelty:**  
First to use dataset splitting (disjoint D1/D2) to prevent memorization in the label correction step. All prior label correction methods (Bootstrapping, M-Correction, DivideMix, SELC) combine assigned labels with the same network's own predictions, which reinforces memorized noise. CrossSplit's peer network cannot have seen the label it is correcting.

**Prior work comparison:**  
Han et al. 2018 (Co-teaching): selects clean samples across two networks — CrossSplit uses soft correction instead of hard selection. Li et al. 2020 (DivideMix): GMM-based clean/noisy split + SSL — CrossSplit bypasses the selection. Karim et al. 2022 (UNICON): contrastive + FixMatch + selection — CrossSplit replaces selection with cross-split correction.

**Verification:**  
KAIST + Mila / Université de Montréal, ICML 2023. Code publicly available.
